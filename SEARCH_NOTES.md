## Why Not Firestore Full-Text Search?

Firestore is an excellent NoSQL document database, but it was **never designed for full-text search**:

1. **No native full-text search** — Firestore supports exact match and range queries only. There is no `LIKE '%keyword%'` equivalent.
2. **No typo tolerance** — A tourist searching "snorkling" (misspelled) would get zero results. This is unacceptable for a booking platform.
3. **No relevance ranking** — Firestore returns results in document order, not by relevance score.
4. **No faceted filtering** — Category counts, price range aggregations, and dynamic filter counts require multiple queries and client-side computation.
5. **Scaling cost** — Querying all documents to filter client-side becomes expensive as the catalog grows.

## How Index Sync Works

The **Firebase Algolia Extension** (`firestore-algolia-search`) automatically syncs specified Firestore collections to Algolia indices:

1. Install the extension from the Firebase Extensions marketplace
2. Configure it to watch `operators` and `activities` collections
3. On any Firestore create/update/delete, the extension auto-syncs to Algolia
4. The `/explore` page queries Algolia directly (not Firestore) for search

### Managing Sync
- **Add new fields to search:** Update the extension config to include new fields
- **Re-index:** Trigger a full re-sync from the Firebase console
- **Monitor usage:** Check Algolia dashboard for search volume and record count
