## Final Decision: Windcave Integration

After reviewing the limitations of Stripe for FJD payouts in Fiji, we have standardized on **Windcave (PxPay 2.0)** as the primary payment gateway.

### Implementation Status
1. **Provider Adapter:** `src/lib/payments/windcave.ts` implements the unified payment interface.
2. **Callback Handler:** `src/app/api/checkout/windcave-callback/route.ts` verifies encrypted responses from PxPay.
3. **Stripe Removal:** All Stripe-related dead code and files have been removed from the repository.

### Key Configuration (Environment Variables)
- `PAYMENT_PROVIDER=windcave`
- `WINDCAVE_USERNAME`
- `WINDCAVE_API_KEY`
- `WINDCAVE_MERCHANT_ID`

### Webhook & Post-Payment
- Webhooks are implemented in `src/app/api/checkout/webhook/route.ts` as a backup.
- Redirect verification is handled in the `windcave-callback` route to ensure immediate UI feedback for the user.
